<a id="sec-3"></a>
## 🧪 阶段3：自动化测试 ← 🎯 当前

> 📅 预计：3-4 周 | "CI/CD 流水线里跑什么？先有测试，才有持续集成的意义。"

- **3.1 测试金字塔与策略**
  - 单元测试 / 集成测试 / E2E 测试的边界
  - 测试覆盖率目标（代码覆盖率 ≠ 质量）
  - 什么该测、什么不该测
- **3.2 PHPUnit 实战**
  - 安装与 phpunit.xml 配置
  - 断言、数据提供器、Mock、Stub
  - 数据库测试（SQLite in-memory / 事务回滚）
- **3.3 进阶测试**
  - 契约测试（Consumer-Driven Contract）
  - 快照测试（Snapshot Testing）
  - 变异测试（Mutation Testing）

### 实践项目

- [ ] 为 `public/index.php` 拆分业务逻辑后编写第一个单元测试
- [ ] 配置 PHPUnit + phpunit.xml
- [ ] 用 Mock 隔离外部依赖（数据库、Redis）
- [ ] 在 CI 流水线中运行测试套件

### 完成标准

- [ ] 能说清单元测试 / 集成测试 / E2E 的分界
- [ ] 能用 Mock 写出不依赖数据库的测试
- [ ] 测试在 GitHub Actions 中自动运行

<a id="sec-advice"></a>
## 💡 学习建议

1. **Windows 用户优先 WSL2** — 代码和 Docker 都跑在 WSL2 里，避免 /mnt/c 跨文件系统 IO 骤降
2. **先会用再深入原理** — 每个阶段先跑通实践，遇到问题再深挖
3. **概念一学一记** — 学完一个子阶段立刻写 concept 文件（复制 `_template.md`），不要攒
4. **踩坑就写答疑** — 实践中的问题记到 concept 的"踩坑记录" + 答疑汇总
5. **不要跳过可观测性** — 日志、监控、追踪是运维的"眼睛"
6. **安全不是最后才学的** — 从 CI/CD 开始就把安全扫描集成到流水线
